# Flux Web Studio — Master Agent Rule
Activation: Always On

This rule defines the operational behaviour, design standards, and build constraints for all agent actions inside this workspace.

The agent must follow this rule when generating pages, content, features, and system architecture.

---

# Project Overview

Project Name:
Flux Web Studio

Domain:
https://fluxwebstudio.com

Project Type:
AI Automation Service Agency

Primary Objective:
Build a professional platform that sells AI automation systems to businesses that want to eliminate repetitive tasks and save time.

Core promise:

"We install AI systems that save businesses 10+ hours per week."

The platform must also be designed to evolve into:

1. Automation template marketplace
2. Digital product ecosystem
3. Flux Automations SaaS platform

All architectural decisions must support this long-term scalability.

---

# Core Services

Flux Web Studio provides three automation services.

The agent must keep services focused and repeatable.

Primary services:

1. Lead Response Automation  
Instantly respond to new website enquiries using AI.

2. Email AI Assistant  
AI reads, summarises, and suggests replies to emails.

3. Client Onboarding Automation  
Automates the onboarding process when a new client signs up.

Services must remain simple, scalable, and productised.

Do not introduce additional services without approval.

The business operates using productised services.

Service packages include:

Starter Automation

The agent must maintain:

• clear service descriptions

# Design System

The design style must follow a modern technology startup aesthetic inspired by companies such as:

Stripe  
Linear  
Reflect  
Notion

The website must look like a credible AI startup, not a freelancer portfolio.

Design principles:

• clean layout  
• strong typography  
• minimal UI elements  
• large spacing between sections  
• professional appearance  
• dark modern interface  

Avoid clutter, gimmicks, and excessive animations.

# Hero Section Design

The homepage hero must feature a **glowing ocean-blue horizon effect** similar to modern AI product landing pages.

The glow represents automation intelligence activating.

The glow should:

• sit at the horizon of the hero section  
• emit a soft ocean-blue gradient light  
• create a futuristic atmosphere  
• animate with subtle pulsing light  

The animation must remain slow and elegant.

Avoid aggressive animations.

# Automation Architecture

Automation systems should prioritise:

reliability  
simplicity  
maintainability  
reusability  

Primary stack:

Make.com  
OpenAI API  
Google Sheets / Airtable / Notion

# Client Workflow

Standard service workflow:

1. Client submits request form
2. Automation requirements reviewed
3. Automation built in Make.com
4. Workflow tested
5. Automation deployed
6. Client receives documentation
7. System saved as reusable template
