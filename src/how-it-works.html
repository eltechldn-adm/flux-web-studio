FAQ

Common Questions

Answers to questions clients typically have before starting an automation project.

How long does an automation project take? +

Most projects are completed within 2–5 working days from request to delivery. A single Starter workflow typically takes 48 hours. More complex multi-workflow projects take up to 7 days. Timelines are included in each service package.

Do I need any technical knowledge to get started? +

No. You only need to describe the workflow you want automated in plain language. We handle all configuration, testing, and technical setup. You do not need to understand how Make.com or the API integrations work.

Can automation connect to my existing tools? +

In most cases, yes. We build with Make.com, which connects to hundreds of common business tools including Gmail, Google Sheets, Airtable, Notion, Slack, CRMs, and more. Let us know your tools during the request and we will confirm compatibility.

What happens if my workflow changes after delivery? +

All automations are built to a reusable template standard, which makes future updates straightforward. If your process changes, you can submit a new request or contact us directly. Modifications are scoped as a separate project.

Is there ongoing support after delivery? +

Each project includes documentation and handover guidance so your team can manage the system independently. For ongoing maintenance or monitoring, we can recommend appropriate options at the point of delivery.
